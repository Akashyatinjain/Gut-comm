2. git clone <repo-url>
Use: Copies an existing remote repository to your local machine.
3. git status
Use: Shows the current state of files (staged, unstaged, untracked).
4. git add <file>
Use: Adds a specific file to the staging area.
5. git add .
Use: Adds all modified and new files to staging.
6. git commit -m "message"
Use: Saves staged changes with a descriptive message.
7. git commit -am "message"
Use: Adds and commits tracked files in one step.
8. git log
Use: Displays commit history.
9. git log --oneline
Use: Shows commit history in a short format.
10. git diff
Use: Shows differences between working directory and staging area.
11. git diff --staged
Use: Shows differences between staged files and last commit.
12. git reset <file>
Use: Removes a file from staging area.
13. git reset --hard
Use: Discards all changes (⚠️ irreversible).
14. git revert <commit-id>
Use: Creates a new commit that undoes a previous commit.
15. git branch
Use: Lists all branches.
16. git branch <branch-name>
Use: Creates a new branch.
17. git checkout <branch-name>
Use: Switches to another branch.
18. git checkout -b <branch-name>
Use: Creates and switches to a new branch.
19. git merge <branch-name>
Use: Merges another branch into current branch.
20. git push origin <branch>
Use: Uploads local commits to remote repository.
